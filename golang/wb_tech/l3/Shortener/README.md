# Shortener

## Описание

Короткие ссылки удобно пересылать в сообщениях, публиковать в интерфейсах и использовать для сбора аналитики. Такой подход нужен маркетологам, блогерам, продуктовым командам и разработчикам, которым важно давать пользователю короткий и понятный адрес.

В этом задании нужно реализовать мини-сервис сокращения ссылок. Он должен генерировать короткие URL, перенаправлять пользователей на оригинальные адреса и собирать статистику по переходам: кто перешел, когда и с какого устройства.

## Основные требования

- Создавать сокращенные ссылки.
- Выполнять редирект по короткому URL.
- Сохранять информацию о переходах в базе данных.
- Поддержать агрегацию аналитики по дням, месяцам и `User-Agent`.

## HTTP API

- `POST /shorten` — создать новую сокращенную ссылку.
- `GET /s/{short_url}` — перейти по короткой ссылке.
- `GET /analytics/{short_url}` — получить аналитику по ссылке.

## Интерфейс

- Форма для создания короткой ссылки.
- Просмотр списка созданных ссылок или результатов сокращения.
- Просмотр аналитики по переходам без ручных `curl`-запросов.

## Дополнительно

- Кэширование популярных ссылок через Redis.
- Поддержка пользовательских алиасов для коротких ссылок.
- Простой UI на любой технологии, включая `HTML + JS` без фреймворков.
