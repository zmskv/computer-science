# EventBooker

## Описание

В сервисах бронирования часто возникает ситуация, когда пользователь занимает место, но не подтверждает или не оплачивает бронь вовремя. Чтобы такие бронирования не блокировали других пользователей, система должна автоматически освобождать место по истечении заданного времени.

В этом задании нужно реализовать сервис бронирования мероприятий, который позволяет создавать события, бронировать места, подтверждать бронь и автоматически отменять неоплаченные бронирования по дедлайну. Это хорошая практика для работы с транзакциями, конкурентным доступом и фоновыми задачами.

## Основные требования

- Создавать мероприятия.
- Бронировать места на мероприятиях.
- Подтверждать или оплачивать бронь.
- Автоматически отменять неоплаченные бронирования по истечении срока.
- Использовать безопасную работу с базой данных, например транзакции, чтобы избежать гонок.
- Реализовать фоновый обработчик через очередь или `cron`-подобный планировщик.

## HTTP API

- `POST /events` — создать мероприятие.
- `POST /events/{id}/book` — забронировать место.
- `POST /events/{id}/confirm` — подтвердить или оплатить бронь.
- `GET /events/{id}` — получить информацию о мероприятии и свободных местах.

## Интерфейс

- Создание мероприятия через форму с названием, датой и количеством мест.
- Просмотр списка мероприятий и их статусов.
- Бронирование места.
- Подтверждение брони.
- Отображение того, что неподтвержденные брони исчезают после заданного времени.
- Разделение на пользовательскую и административную части, даже если интерфейс остается простым.

## Дополнительно

- Уведомления об отмене бронирования через `Email` или `Telegram`.
- Поддержка нескольких пользователей и учет их бронирований.
- Настройка разного срока жизни брони для разных мероприятий.
