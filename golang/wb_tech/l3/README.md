# L3

## О разделе

В этом разделе предлагается с нуля создать несколько приложений с учетом того, что бизнес-требования могут меняться, как и технологический стек. При выполнении заданий важно учитывать эту гибкость и закладывать ее в архитектурные решения.

## Общее требование

- Во всех проектах нужно использовать [фреймворк `wbf`](https://github.com/wb-go/wbf), оставшийся от расформированной команды.

## Проекты

- [DelayedNotifier](DelayedNotifier/README.md) — сервис отложенных уведомлений через очереди.
- [Shortener](Shortener/README.md) — сервис сокращения URL с аналитикой переходов.
- [SalesTracker](SalesTracker/README.md) — CRUD-сервис с аналитикой и агрегированием данных.
- [CommentTree](CommentTree/README.md) — сервис древовидных комментариев с поиском и навигацией.
- [WarehouseControl](WarehouseControl/README.md) — складской CRUD с историей изменений и ролевой моделью.
- [EventBooker](EventBooker/README.md) — сервис бронирования с дедлайнами и автоотменой.
- [ImageProcessor](ImageProcessor/README.md) — сервис фоновой обработки изображений через очередь.
