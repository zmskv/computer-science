# WarehouseControl

## Описание

Склад — это не просто список товаров, а постоянно меняющаяся система, в которой важно понимать, кто, что, когда и зачем изменил. При этом не все сотрудники должны иметь одинаковые права: кладовщик может редактировать данные, менеджер — только просматривать, а аудитор — анализировать историю изменений.

В этом задании нужно реализовать мини-систему учета товаров на складе с базовыми CRUD-операциями, журналом изменений и ролевой моделью доступа. Это типичный пример корпоративного backend-сервиса для логистики, производства и торговли.

Отдельное требование задания — реализовать историю изменений через триггеры внутри СУБД. Это намеренно выбранный антипаттерн: он нужен как учебный пример того, как делать не стоит в реальных проектах.

## Основные требования

- Реализовать CRUD-операции для инвентаря.
- Хранить историю изменений данных: кто, когда и что изменил.
- Поддержать роли пользователей, например:
  - `admin` — полный доступ;
  - `manager` — просмотр и редактирование;
  - `viewer` — только просмотр.
- Разграничить права доступа по ролям.
- Использовать триггеры в базе данных для сохранения истории изменений.
- Реализовать авторизацию через JWT: роль должна передаваться в токене и проверяться при каждом запросе.

## HTTP API

- `POST /items` — создать товар.
- `GET /items` — получить список товаров.
- `PUT /items/{id}` — обновить товар.
- `DELETE /items/{id}` — удалить товар.

## Интерфейс

- Войти от имени пользователя с выбранной ролью.
- Просматривать список товаров.
- Добавлять, редактировать и удалять товары при наличии прав.
- Просматривать историю изменений по каждому товару в табличном виде.

## Дополнительно

- Просмотр отличий между версиями товара.
- Поиск и фильтрация истории изменений по дате, пользователю и действию.
- Экспорт истории в CSV.
