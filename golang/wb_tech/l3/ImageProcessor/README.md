# ImageProcessor

## Описание

Сервисы с загрузкой изображений встречаются повсюду: в соцсетях, маркетплейсах, фотогалереях, блогах и редакторах. Чтобы изображения быстро и корректно отображались в разных сценариях, обычно требуется фоновая обработка: создание миниатюр, уменьшенных копий, версий с водяными знаками и других производных файлов.

В этом задании нужно реализовать сервис, который принимает изображения, отправляет задачи на обработку в очередь через Apache Kafka и выполняет обработку в фоне. Такой подход позволяет не блокировать пользователя и лучше масштабировать систему.

## Основные требования

- Принимать изображения на обработку.
- Использовать Apache Kafka для очереди фоновых задач.
- Поддержать фоновую обработку изображений:
  - изменение размера;
  - генерация миниатюр;
  - добавление водяных знаков.
- Хранить исходные и обработанные изображения в отдельном файловом хранилище.

## HTTP API

- `POST /upload` — загрузить изображение на обработку.
- `GET /image/{id}` — получить обработанное изображение.
- `DELETE /image/{id}` — удалить изображение.

## Интерфейс

- Загрузка изображения через форму с `<input type="file">`.
- Просмотр статуса обработки, например через состояние «в обработке».
- Получение готового результата и его отображение на странице.
- Удаление изображения.
- Один экран со списком загруженных изображений и их состоянием.

## Дополнительно

- Поддержка нескольких форматов обработки, например `jpg`, `png` и `gif`.
