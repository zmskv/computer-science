# SalesTracker

## Описание

Во многих сервисах недостаточно просто хранить записи о продажах, транзакциях или событиях: по ним еще нужно строить аналитику. Обычно требуется считать сумму продаж за период, количество операций, средний чек, медиану, перцентили и другие агрегаты.

В этом задании нужно реализовать сервис, который сочетает CRUD-операции с аналитикой по данным. Это хорошая практика для работы с SQL-запросами, агрегацией и обработкой бизнес-метрик.

Все данные должны храниться в PostgreSQL. Для агрегации, фильтрации, расчета медианы и перцентилей нужно использовать SQL-запросы, например с оконными функциями или подзапросами. Данные желательно валидировать, а запросы — защищать от SQL-инъекций.

## Основные требования

- Реализовать CRUD-операции для записей, например продаж или финансовых транзакций.
- Хранить данные в PostgreSQL.
- Поддержать агрегированную аналитику:
  - сумма (`sum`);
  - среднее (`avg`);
  - количество (`count`);
  - медиана;
  - 90-й перцентиль.
- Добавить фильтрацию аналитики по периоду.
- Валидировать входные данные, например сумму и дату.

## HTTP API

- `POST /items` — создать запись.
- `GET /items` — получить список записей.
- `PUT /items/{id}` — обновить запись.
- `DELETE /items/{id}` — удалить запись.
- `GET /analytics?from={from}&to={to}` — получить агрегированную аналитику за период.

## Интерфейс

- Добавлять новые записи, например доходы и расходы.
- Просматривать таблицу всех записей с фильтрами.
- Получать сводную аналитику за период в виде чисел или графика.
- Скачивать данные в CSV, если экспорт реализован.

## Дополнительно

- Экспорт отчетов в CSV.
- Сортировка по различным полям.
- Группировка по дням, неделям, категориям и другим срезам.
