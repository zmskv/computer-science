# DelayedNotifier

## Описание

Во многих сервисах уведомления нужно отправлять не сразу, а через заданное время. Это могут быть напоминания о встречах, сообщениях, заказах или других событиях. Такие задачи обычно выполняет отдельный сервис, который умеет принимать запросы, ждать нужный момент и отправлять сообщение автоматически.

В этом задании нужно реализовать сервис отложенных уведомлений. Он должен принимать запросы на создание уведомлений, складывать их в очередь через RabbitMQ, а затем отправлять в указанное время. Если отправка завершилась ошибкой, сервис должен повторить попытку позже.

## Основные требования

- Принимать уведомления с датой и временем отправки.
- Использовать RabbitMQ для очереди уведомлений.
- Отправлять уведомления в указанное время.
- Повторять отправку при ошибке с экспоненциальной задержкой.

## HTTP API

- `POST /notify` — создать уведомление с датой и временем отправки.
- `GET /notify/{id}` — получить статус уведомления.
- `DELETE /notify/{id}` — отменить запланированное уведомление.

## Интерфейс

- Форма для создания уведомления.
- Просмотр списка текущих уведомлений.
- Отображение статусов уведомлений.

## Дополнительно

- Использование Redis для кэширования и быстрой проверки статуса.
- Поддержка нескольких каналов доставки, например `Email` и `Telegram`.
- Простой UI на любой технологии, включая `HTML + JS` без фреймворков.
